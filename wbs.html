1. Demo

A minimal working prototype connecting a solver to the plant3D which export a simple scene and connects them using specific pipes

1.1. stage one

a simple scene with only two components each of which has only one input. This stage requires almost no setup in plant3D and it's the fastest way of achiving proof of concept.

1.1.1. document manual process involve in this scenario

  1. manually export components and their inputs into a excel file
  2. manually import this PCF structure into plant3D

1.1.2. define gRPC routing request/response interface   design

1.1.3. implement a mock solver which returns specific response

1.1.4. implement simple solver

  1. solver implementation in any possible language
  2. implement solver gRPC stubs to communicate with server

1.1.5. a simple gRPC server which handles communication between solver

  1. setup gRPC server/client stubs
  2. convert the export excel into a routing request
  3. communicate with solver to get the routing response
  4. convert the roating response into PCF format

1.2. stage two

this stage requires very basic understanding and set up in plant3D, however it's the foundation of furthure development. here we want very simple plugin that eithre using a simple time or a simple button in the screen dose the import and export of the pcf

1.2.1. set up a minimal plant3D

  1. install a working version of plant3D on windows
  2. setup Visual studio to develop plant3D plugin
  3. setup required SKD in visual studio
  4. write a hello word plugin in plant3D

1.2.2. write a simple plugin that comminicates with the server that is developed in previous stage

  1. write a simple component that export component list into excel file in specific folder
    1. find the appropriate documentation regarding component export in SDK
    2. define proper interface for the component
    3. component implementation
  2. write a simple component that automatically imports PCF from specific folder
    1. find the appropriate documentation
    2. define proper interface for the component
    3. component implementation
  3. write a simple UI with two buttons to export components and import PCF from server
    1. Research about creating UI window and elements and event handling
    2. Implement the popup window and connect it to the components

1.3. stage three

the goal of this stage is to provide improved UI/UX for the user and this the begining stage of actaully providing a proper plugin that can be given to customers. in this stage a series of module for proper architecture and separation of tasks are introduced.

1.3.1. define the basic workflow of the user

goal here is to specify how the user is going to interact with our system, for instance we could ask the user to chose the starting point and click on a button then select the target and click on a button then press an other button to view a preview of the routing. This task requires some UI/UX design

1.3.2. implement export module

  1. design export module interfaces
    1. interface for exporting scene components
    2. interface for exporting component information
    3. interface for exporting component exclusion or proximity zones
    4. interface for exporting exclusion zones of the scene
    5. interface for checking collisions with the exclusion zones
    6. interface for listing all the objects in the scene
    7. interface for interrogating an object

1.3.3. implement scene interaction module

this module is supposed to separate object and scene manipulation actions from the rest of the plugin

  1. design scene interaction module interface
    1. interface to update an object property
    2. interface to move/rotate objects
    3. …

1.3.4. implement user interface module

  1. research and document plant3D user interface
  2. design user interface elements and signals for user interface module
  3. design module interface

    this supposed to separate detail implementation of plant3D SDK from the rest of the plugin
